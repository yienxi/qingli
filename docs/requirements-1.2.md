# 轻历 1.2 迭代需求文档

## 产品定位

轻历 1.2 的主定位是轻量日历查询工具。核心价值是让用户快速查询公历、农历、节气、节假日、调休和宜忌信息。趣味文案、今日卡片、分享卡片和轻动效是增强留存与传播的补充，不改变日历查询的主线。

## 迭代目标

1. 强化月历作为页面主视觉和核心交互区。
2. 保留左侧今日卡片，但从杂志内容卡收敛为“日期摘要辅助区”。
3. 统一页面字号、间距、圆角、状态色和交互反馈。
4. 增加克制的氛围动效，提升质感但不干扰查询。
5. 将 AI、iCal、PWA 等能力降级为辅助配置，不抢首屏注意力。
6. 保持零注册、打开即用、本地事件存储的产品边界。

## 信息架构

桌面端采用左右结构：

- 左侧：今日/所选日期卡片，占比约 28%-32%。
- 右侧：月历查询区，占比约 68%-72%。
- 下方：订阅与辅助能力，默认弱化展示。

移动端采用纵向结构：

- 今日/所选日期卡片。
- 月历查询区。
- 日期详情面板。
- 订阅与辅助能力。

## 今日卡片规范

今日卡片展示以下信息：

- 公历日期、星期、农历日期。
- 一句轻趣味摘要，基于当天日期与宜忌生成。
- 今日宜，最多 3-4 个。
- 今日忌，最多 3-4 个。
- 节气、节日、调休、冲煞等提醒，有则展示，无则不占位。
- 分享今日轻历卡按钮。

今日卡片不默认展示值神、纳音、星宿、喜神、财神等深层传统字段。这些信息保留到日期详情面板。

## 月历查询规范

月历是核心区域：

- 月份标题清晰，日期数字优先级最高。
- 农历、节气、节假日作为副信息展示。
- 今天、选中日期、周末、节假日、调休状态必须有清晰区分。
- 点击任意日期后，左侧今日卡片更新为所选日期卡片。
- 所选非今天日期时，卡片提供回到今天入口。
- 切换月份时日期格轻微淡入，不做夸张动效。

## 趣味内容规则

趣味内容来自日期本身，而不是业务场景选择：

- 星期节奏。
- 节气或传统节日。
- 是否周末、临近周末或调休。
- 传统宜忌映射为现代口语。

生成规则：

- 同一日期生成固定内容。
- 刷新页面不随机变化。
- 切换日期才更新。
- AI 摘要默认关闭，只作为可选增强。

## 动效规范

动效用于提升质感和时间感，不抢日历查询注意力。

第一阶段实现：

- 背景极淡粒子层。
- 今日卡片淡入。
- 日期格 hover 与选中微交互。
- 月份切换时日期格 stagger 淡入。

约束：

- 移动端降低粒子数量。
- 遵守 `prefers-reduced-motion`。
- 不使用高密度粒子、强视差、霓虹动效、大面积浮动装饰。

## 配置规范

新增前端配置文件，集中管理首页展示和增强能力：

```js
{
  aiSummary: false,
  ambientMotion: true,
  icalSectionCollapsed: true,
  pwaBanner: 'deferred',
  todayCardMode: 'date-summary',
  personalEvents: true
}
```

## 第一阶段交付范围

1. 新增 `docs/requirements-1.2.md`。
2. 新增 `js/config.js`，集中配置 AI、动效、订阅区和 PWA 行为。
3. 新增 `js/ambient.js`，实现低干扰背景粒子。
4. 重构今日卡片文案与结构，移除过重的生肖杂志感和默认 AI 主导。
5. 点击日期后更新左侧卡片，使左卡和月历形成闭环。
6. 优化月历视觉权重和日期格交互。
7. 将 iCal 订阅区默认折叠。

## 非目标

- 不开放注册。
- 不做复杂个人日程系统。
- 不做业务场景选择器。
- 不让 AI 成为默认首屏内容来源。
- 不把轻历改成内容资讯站或营销落地页。
